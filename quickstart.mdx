## Prerequisites

Before you start, make sure you have:

- **Nix** installed with flakes support (`nix --version` should work)
- **Node.js >= 18** for the `npx` launcher
- **KVM access** on Linux hosts — `/dev/kvm` must be readable and writable by your user
- A supported platform: Linux `x86_64` or `aarch64`, or Apple Silicon macOS (experimental)

<Note>
On Apple Silicon macOS, KVM is not required. The launcher detects this automatically. macOS support is experimental — please [report issues](https://github.com/skadaai/firebreak/issues) if you encounter them.
</Note>

<Warning>
Intel Macs are not supported. Firebreak requires Apple Silicon (arm64) on macOS.
</Warning>

## Get started

<Steps>
  <Step title="Install Firebreak">
    You can run Firebreak directly with `npx` — no global install required:

    ```bash
    npx firebreak --help
    ```

    Or launch a VM workload directly with Nix:

    ```bash
    nix run github:skadaai/firebreak#firebreak -- --help
    ```

    If you're working inside a cloned Firebreak repository, the launcher detects that automatically and uses your local checkout instead of fetching from GitHub.
  </Step>

  <Step title="Initialize your project">
    Run `firebreak init` in your project directory. It will ask a few questions and write a `.firebreak.env` file with your preferred defaults:

    ```bash
    npx firebreak init
    ```

    The interactive prompts ask you to choose:
    - **Agent config mode**: `workspace` (recommended), `vm`, `host`, or `fresh`
    - **Default launch mode**: `run` (launches the agent) or `shell` (maintenance shell)
    - Optional per-agent overrides for Codex and Claude Code

    To skip the prompts and write a template with all options commented out:

    ```bash
    npx firebreak init --non-interactive
    ```

    The resulting `.firebreak.env` looks like:

    ```dotenv .firebreak.env
    AGENT_CONFIG=workspace
    # FIREBREAK_LAUNCH_MODE=run
    # CODEX_CONFIG=workspace
    # CLAUDE_CONFIG=workspace
    ```

    Real environment variables always take precedence over values in this file.
  </Step>

  <Step title="Launch an agent VM">
    Start Codex or Claude Code in an isolated VM:

    <CodeGroup>
    ```bash Codex
    npx firebreak run codex
    ```
    ```bash Claude Code
    npx firebreak run claude-code
    ```
    </CodeGroup>

    On first run, Firebreak fetches the VM package from GitHub (or uses your local checkout). Subsequent runs use the Nix store cache and start much faster.

    Your current working directory is automatically mounted into the VM at the same path. The agent can read and write your project files, but cannot affect the host system outside that directory.

    To enter the maintenance shell instead of launching the agent:

    ```bash
    npx firebreak run codex --shell
    ```
  </Step>

  <Step title="Verify your setup with doctor">
    At any point, run `firebreak doctor` to see your full resolved configuration:

    ```bash
    npx firebreak doctor
    ```

    This reports your project root, resolved config file, platform, KVM state, agent config modes, and current working directory compatibility. Use `--json` for machine-readable output:

    ```bash
    npx firebreak doctor --json
    ```
  </Step>
</Steps>
