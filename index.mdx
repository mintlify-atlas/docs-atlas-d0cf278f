Skada Firebreak gives you a small, clear interface for running AI coding agents like Codex and Claude Code inside isolated MicroVMs. Each agent session gets its own KVM-backed sandbox with your project directory mounted in, so agents can write and run code without touching the host system.

## How it works

Firebreak wraps AI agent CLIs in lightweight MicroVMs using `microvm.nix` and KVM. Your current working directory is mounted into the VM at the same path, so agents see your project exactly as it exists on the host. When the agent exits, the VM tears down.

<Steps>
  <Step title="Install Firebreak">
    Run `npx firebreak init` in your project directory. Firebreak checks that Nix and KVM are available, then writes a `.firebreak.env` config file tailored to your answers.
  </Step>
  <Step title="Launch an agent VM">
    Run `npx firebreak run codex` or `npx firebreak run claude-code` to start an agent in an isolated VM. Your project directory is mounted automatically.
  </Step>
  <Step title="Work with confidence">
    The agent runs inside KVM isolation. It can read and modify your project files, but it cannot affect the host system outside the mounted workspace.
  </Step>
</Steps>
