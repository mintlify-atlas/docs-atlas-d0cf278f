title: "`firebreak init`"
description: "Interactively write a `.firebreak.env` project config file with prompts for config mode, launch mode, and per-agent overrides."
---

## Synopsis

```bash
firebreak init [--force] [--stdout] [--interactive] [--non-interactive]
```

## What it does

`firebreak init` writes a `.firebreak.env` file into the current project root. This file stores project-level defaults for config mode, launch mode, and optional per-agent overrides. Real environment variables always take precedence over values in the file.

In interactive mode (the default), Firebreak prompts you for each setting and shows a preview of the generated file before writing it. In non-interactive mode, it writes a commented-out template with all options present but inactive.

## Flags

| Flag | Description |
|---|---|
| `--force` | Overwrite an existing `.firebreak.env` without prompting. Without this flag, `init` exits with an error if the file already exists. |
| `--stdout` | Print the generated config to stdout instead of writing it to a file. Compatible with both interactive and non-interactive modes. |
| `--interactive` | Prompt for answers interactively. This is the default behavior. |
| `--non-interactive` | Write a template with all options commented out. No prompts are shown. |

<Note>
If a `.firebreak.env` already exists and you run `firebreak init` without `--force`, the command exits with an error and suggests running `firebreak doctor` to inspect the current resolved state.
</Note>

## Interactive flow

When run interactively, `firebreak init` walks through the following prompts:

<Steps>
  <Step title="Shared default config mode">
    Choose how all agent VMs locate their config directories by default.

    | Option | Behavior |
    |---|---|
    | `workspace` (default) | Use the project-local config directory when present |
    | `vm` | Keep config inside the VM across launches |
    | `host` | Use a shared host-side config directory |
    | `fresh` | Start from an empty runtime config each launch |
  </Step>
  <Step title="Default local launch mode">
    Choose what happens when you run `firebreak run <vm>` without any launch mode flag.

    | Option | Behavior |
    |---|---|
    | `run` (default) | Launch the agent directly |
    | `shell` | Enter the VM maintenance shell instead |
  </Step>
  <Step title="Host config path (if host mode selected)">
    If you chose `host` in step 1, you are prompted for the shared host-side config directory path. The default is `~/.config/firebreak-agent`.

    Paths with whitespace are not accepted, as the local VM wrapper does not support them.
  </Step>
  <Step title="Optional Codex-specific override">
    Optionally add a Codex-specific config mode that overrides the shared default. If you select `host` mode for Codex, you are prompted for the Codex host config path (default: `~/.codex`).
  </Step>
  <Step title="Optional Claude Code-specific override">
    Optionally add a Claude Code-specific config mode that overrides the shared default. If you select `host` mode for Claude Code, you are prompted for the Claude Code host config path (default: `~/.claude`).
  </Step>
  <Step title="Confirm and write">
    Firebreak shows a preview of the generated `.firebreak.env` and asks you to confirm before writing. If you decline, the command exits without creating or modifying any files.
  </Step>
</Steps>

## Example output

A typical `.firebreak.env` generated by `firebreak init`:

```dotenv
# Firebreak project defaults
#
# Generated by 'firebreak init'.
# Real environment variables override values in this file.

AGENT_CONFIG=workspace
FIREBREAK_LAUNCH_MODE=run
```

With a Codex host-mode override added:

```dotenv
# Firebreak project defaults
#
# Generated by 'firebreak init'.
# Real environment variables override values in this file.

AGENT_CONFIG=workspace
FIREBREAK_LAUNCH_MODE=run

CODEX_CONFIG=host
CODEX_CONFIG_HOST_PATH=~/.codex
```

The non-interactive template (written by `firebreak init --non-interactive`):

```dotenv
# Firebreak project defaults
#
# Real environment variables override values in this file.

# Shared config mode for local workload VMs:
AGENT_CONFIG=workspace

# Public local launch selector:
# FIREBREAK_LAUNCH_MODE=run
# FIREBREAK_WORKER_MODE=local
# FIREBREAK_WORKER_MODES=codex=vm,claude=local

# Optional shared host config root when AGENT_CONFIG=host:
# AGENT_CONFIG_HOST_PATH=~/.config/firebreak-agent

# Optional per-agent overrides:
# CODEX_CONFIG=workspace
# CODEX_CONFIG_HOST_PATH=~/.codex
# CLAUDE_CONFIG=workspace
# CLAUDE_CONFIG_HOST_PATH=~/.claude
```

## Environment variable precedence

Values set in `.firebreak.env` are applied as project defaults. Any matching environment variable already present in the shell takes precedence and the file value is ignored. Use `firebreak doctor` to see the fully resolved state.

## Examples

```bash
# Interactive init (default)
firebreak init

# Overwrite an existing config
firebreak init --force

# Preview what would be written without touching the filesystem
firebreak init --stdout

# Write a commented template without any prompts
firebreak init --non-interactive

# Preview the non-interactive template on stdout
firebreak init --non-interactive --stdout
```
